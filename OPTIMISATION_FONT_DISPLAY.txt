OPTIMISATION FONT-DISPLAY: SWAP
Date: 2025-01-27
Objectif: Corriger l'avertissement PageSpeed concernant font-display

PROBLÈME IDENTIFIÉ:
- Avertissement PageSpeed: "Envisagez de définir font-display sur swap ou optional"
- Économies estimées: 20 ms
- Les polices Google Fonts étaient chargées via @import dans le CSS, ce qui bloque le rendu

SOLUTION APPLIQUÉE:

1. REMPLACEMENT DES @IMPORT PAR PRELOAD
   - Tous les @import url('https://fonts.googleapis.com/...') ont été remplacés par des <link rel="preload"> avec onload
   - Utilisation de font-display: swap dans toutes les URLs Google Fonts
   - Ajout de preconnect pour optimiser les connexions

3. TECHNIQUE UTILISÉE:

   Avant (bloquant):
   ```css
   @import url('https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700;800;900&display=swap');
   ```

   Après (non-bloquant):
   ```html
   <link rel="preconnect" href="https://fonts.googleapis.com" crossorigin>
   <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
   <link rel="preload" href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700;800;900&display=swap" as="style" onload="this.onload=null;this.rel='stylesheet'">
   <noscript><link href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700;800;900&display=swap" rel="stylesheet"></noscript>
   ```

   + Force font-display: swap dans le CSS:
   ```css
   @font-face {
       font-family: 'Inter';
       font-display: swap;
   }
   ```

4. AVANTAGES:
   - Élimination du blocage du rendu causé par @import
   - font-display: swap garantit que le texte est toujours visible
   - Preload améliore les performances de chargement
   - Preconnect réduit la latence DNS
   - Économie estimée: 20 ms (selon PageSpeed)
